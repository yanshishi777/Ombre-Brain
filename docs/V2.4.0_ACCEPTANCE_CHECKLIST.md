# OmbreBrain v2.4.0 Pre-Release Acceptance Checklist

This checklist is for local validation before any public upload. Passing it does not mean publish automatically.

## Local Verification

- [ ] Run focused final tests:

```powershell
py -3.10 -m pytest tests\test_v3_architecture_audit.py tests\test_v3_resilience_scanner.py tests\test_v3_maintenance_report.py tests\test_v3_architecture_docs.py tests\test_v3_release_acceptance.py -q
```

- [ ] Run all architecture tests:

```powershell
$files = @(Get-ChildItem -Path tests -Filter 'test_v3_*.py' | ForEach-Object { $_.FullName }); py -3.10 -m pytest @files -q
```

- [ ] Run the full suite:

```powershell
py -3.10 -m pytest -q
```

- [ ] Generate a local health report:

```powershell
py -3.10 tools/v3_health_report.py --buckets-dir buckets --output reports/v3-health.json
```

## Compatibility Checks

- [ ] MCP tool names unchanged.
- [ ] MCP arguments and return text unchanged.
- [ ] bucket markdown unchanged.
- [ ] Dashboard existing routes unchanged.
- [ ] Config/env names keep their current meaning.
- [ ] Hot update protection still excludes config, buckets, vector database data, OAuth secrets, and deployment user overrides.
- [ ] Decision debug and maintenance paths remain read-only for legacy state.

## Publication Guardrails

- [ ] No commit.
- [ ] No push.
- [ ] No release.
- [ ] No GitHub tag.
- [ ] No package publication.
- [ ] No cloud server change.
- [ ] No SSH or remote server operation.

## Manual Review

- [ ] Review `docs/V2.4.0_ARCHITECTURE.md`.
- [ ] Review `docs/V2.4.0_BOUNDARY_MAP.md`.
- [ ] Review `docs/V2.4.0_ROLLBACK.md`.
- [ ] Review `docs/V2.4.0_RELEASE_NOTES_DRAFT.md`.
- [ ] Review generated artifacts and diff before uploading anything by hand.

## Rollback

If any acceptance item fails, follow `docs/V2.4.0_ROLLBACK.md`. Rollback should favor removing local v2.4.0 side-channel state or reverting local code before publication, never changing production buckets or vector data as a first response.
